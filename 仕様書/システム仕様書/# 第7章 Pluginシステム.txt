# 第7章 Pluginシステム

## 7.1 概要

Pluginシステムは、本ソフトの機能をゲーム本体とは独立して拡張するためのシステムである。

ゲーム本体のコードを変更することなく、新しい機能・見た目・データを追加できる設計とする。

PluginはJSONを中心としたデータ構成を基本とし、将来的にはJavaScript等による機能拡張にも対応できる構造を想定する。

---

# 7.2 基本方針

Pluginシステムは以下の方針で設計する。

- ゲーム本体から独立する
- 有効・無効を切り替えられる
- 個別に導入・削除できる
- Plugin単位で管理する
- バージョン管理に対応する
- 他Pluginへ極力依存しない
- 将来の拡張を考慮する

---

# 7.3 Pluginの種類

本システムでは以下のPluginを想定する。

- Pattern Plugin
- Skill Plugin
- Theme Plugin
- Sound Plugin
- Map Plugin
- Appearance Plugin
- Language Plugin
- Effect Plugin
- UI Plugin

将来的には追加できる設計とする。

---

# 7.4 Pattern Plugin

Patternを追加するPlugin。

追加できる内容

- Pattern
- Pattern Pack
- Pattern Tree
- Patternタグ

ゲーム起動時に読み込む。

---

# 7.5 Skill Plugin

新しいSkillを追加するPlugin。

追加できる内容

- 検索条件
- 比較方法
- フィルター
- UI
- Skillプリセット

---

# 7.6 Theme Plugin

ゲーム全体の見た目を変更する。

変更対象

- UI
- ボタン
- フォント
- ウィンドウ
- 色
- 背景

ゲーム本体とは独立する。

---

# 7.7 Sound Plugin

追加できる内容

- BGM
- 効果音
- UI音
- Flow演出音
- コンボ演出音

---

# 7.8 Map Plugin

マップの見た目を変更する。

例

- 森
- ダンジョン
- 街
- サイバー
- 和風

ゲームルールは変更しない。

---

# 7.9 Appearance Plugin

表示アイコンを追加する。

対象

- ファイル
- フォルダ
- UIアイコン
- エフェクト

拡張子アイコンの追加にも対応する。

---

# 7.10 Language Plugin

表示言語を追加する。

対象

- UI
- メニュー
- 説明文
- ログ

ゲーム本体の変更は不要とする。

---

# 7.11 Effect Plugin

演出を追加する。

対象

- パーティクル
- Flow
- コンボ
- Pattern Burst
- カメラ演出

ゲームバランスには影響しない。

---

# 7.12 UI Plugin

UIの追加・変更を行う。

例

- 新しい情報パネル
- HUD
- メニュー
- ログ表示

既存UIとの共存を前提とする。

---

# 7.13 Plugin構成

Pluginはフォルダ単位で管理する。

例

Plugin/

PluginName/

manifest.json

README.md

Data/

Assets/

Config/

必要に応じて追加ファイルを持てる。

---

# 7.14 Manifest

すべてのPluginは

manifestを持つ。

管理内容

- Plugin ID
- 名前
- 作者
- バージョン
- 対応Version
- 説明
- 種類
- タグ

ゲーム起動時に読み込む。

---

# 7.15 Plugin読込

ゲーム起動時

Pluginフォルダを検索する。

↓

manifest読込

↓

Version確認

↓

有効Pluginのみ読込

↓

ゲーム開始

ゲーム中の追加読込は行わない。

次回起動時に反映する。

---

# 7.16 Plugin管理画面

設定プロジェクトから管理する。

機能

- 一覧表示
- 検索
- 有効
- 無効
- 詳細表示

ゲーム本体では設定のみ利用する。

---

# 7.17 Plugin競合

同じ内容を変更するPluginが存在する場合、

優先順位を使用する。

優先順位が同じ場合、

プレイヤーへ通知する。

自動上書きは行わない。

---

# 7.18 Pluginエラー

以下の場合

Pluginを読み込まない。

- Manifest破損
- JSON破損
- Version非対応
- 必須項目不足

エラーはログへ記録する。

ゲームは継続する。

---

# 7.19 Plugin更新

Pluginは

差し替えによる更新を想定する。

Version変更時は

互換性を確認する。

---

# 7.20 Plugin削除

Plugin削除時

保存データは削除しない。

Pluginを再導入すると

以前の設定を利用できる。

---

# 7.21 Plugin依存

基本的に

Plugin同士の依存は持たない。

将来的に必要になった場合は

Manifestで依存関係を定義できるよう設計する。

---

# 7.22 Plugin保存

Plugin固有の設定は

Plugin専用フォルダへ保存する。

ゲーム本体とは分離する。

---

# 7.23 Pluginログ

Plugin関連のログを保存する。

例

- 読込
- 更新
- エラー
- Version確認
- 無効化

ログフィルターに対応する。

---
