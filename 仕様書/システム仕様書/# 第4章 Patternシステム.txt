# 第4章 Patternシステム

## 4.1 概要

Patternシステムは、プレイヤーの整理ルールを管理・再利用するためのシステムである。

PatternはAIによる自動分類ではなく、プレイヤーの整理方法を保存し、次回以降の整理作業を支援することを目的とする。

PatternはJSON形式で保存し、ゲーム起動時に読み込む。

---

# 4.2 Patternとは

Patternとは、

「特定の条件に一致したファイルを、指定した整理フォルダへ整理するためのルール」

である。

Patternはプレイヤー自身が作成・編集・共有できる。

ゲームはPatternを提案するのみであり、自動で整理を決定しない。

---

# 4.3 Patternの構成

Patternは以下の情報を保持する。

- Pattern ID
- 名前
- 説明
- バージョン
- 作成者（任意）
- 作成日時
- 更新日時
- 有効 / 無効
- 優先度
- タグ
- 条件一覧
- 整理先フォルダ
- 使用回数
- 成功回数
- 成功率

---

# 4.4 Pattern条件

Patternは複数条件を持つことができる。

対応予定条件

- 拡張子
- ファイル名一致
- 部分一致
- 前方一致
- 後方一致
- 正規表現
- 作成日
- 更新日
- サイズ
- 親フォルダ
- ファイル属性
- カスタム条件（Plugin）

---

# 4.5 条件の組み合わせ

Patternは複数条件を組み合わせられる。

対応する条件

- AND
- OR
- NOT

複雑な整理ルールにも対応できる設計とする。

---

# 4.6 Pattern優先度

複数のPatternが一致した場合、

Priority（優先度）の高いPatternを優先する。

優先度が同じ場合は、

プレイヤーへ候補一覧を表示する。

自動決定は行わない。

---

# 4.7 Patternタグ

Patternにはタグを設定できる。

例

- 画像
- 動画
- ゲーム
- Unity
- 仕事
- 一時ファイル
- バックアップ

タグは検索や分類に利用する。

---

# 4.8 Pattern状態

Patternは以下の状態を持つ。

- 有効
- 無効

無効なPatternは読み込むが、

候補表示には利用しない。

---

# 4.9 Pattern作成

Patternは以下の方法で作成できる。

- 手動作成
- ゲーム中に作成
- 整理履歴から作成
- インポート

---

# 4.10 Pattern編集

編集可能項目

- 名前
- 条件
- タグ
- 優先度
- 整理先
- 説明
- 状態

編集内容はJSONへ保存する。

次回起動時に反映する。

---

# 4.11 Pattern削除

Patternは削除できる。

削除前に確認ダイアログを表示する。

削除後は復元できない。

---

# 4.12 Pattern検索

以下の条件で検索できる。

- 名前
- タグ
- 整理先
- 条件
- 作成日
- 更新日
- 使用回数

---

# 4.13 Pattern推薦

ゲームは整理状況を分析し、

利用できそうなPatternを提案する。

Patternは候補として表示するだけであり、

プレイヤーの確認なしに適用しない。

---

# 4.14 Pattern学習

プレイヤーが同じ整理を繰り返した場合、

Patternとして保存することを提案する。

Patternを自動生成しても、

保存するかどうかはプレイヤーが決定する。

---

# 4.15 Pattern Import

Patternは外部JSONから読み込める。

読み込み時は

- バージョン
- JSON形式
- 重複

を確認する。

---

# 4.16 Pattern Export

任意のPatternをJSONとして出力できる。

出力したPatternは他ユーザーと共有できる。

---

# 4.17 Pattern Pack

複数PatternをまとめたものをPattern Packと呼ぶ。

例

- Unity整理
- 写真整理
- 動画整理
- イラスト整理

Pattern Pack単位で

Import・Exportが可能。

---

# 4.18 Pattern Marketplace

将来的にオンライン共有へ対応できる設計とする。

共有内容

- Pattern
- Pattern Pack

ゲーム本体とは独立したデータとして管理する。

---

# 4.19 Pattern Tree

Patternは階層表示へ対応する。

例

画像

├ JPG

├ PNG

└ GIF

動画

├ MP4

├ AVI

└ MKV

表示方法は自由に変更できる。

---

# 4.20 Pattern統計

Patternごとに以下を保存する。

- 使用回数
- 成功回数
- 成功率
- 最終使用日時
- 平均整理件数

統計はリザルト画面でも利用する。

---

# 4.21 Patternキャッシュ

ゲーム起動時に全Patternを読み込む。

ゲーム中はキャッシュを利用する。

Patternの追加・変更・削除は、

次回起動時に反映する。

---

# 4.22 JSON管理

Patternは1ファイルごとに保存する。

例

Patterns/

├ Image.json

├ Video.json

├ Unity.json

├ Work.json

└ Screenshot.json

Pattern同士は独立して管理する。

---

# 4.23 Pattern ID

すべてのPatternは一意のIDを持つ。

参照は名前ではなくPattern IDで行う。

---

# 4.24 エラー処理

以下の場合はPatternを読み込まない。

- JSON破損
- バージョン非対応
- 必須項目不足
- 条件エラー

エラー内容はログへ記録する。

---

# 4.25 将来拡張

将来的に以下へ対応可能な設計とする。

- クラウド同期
- オンライン共有
- Pattern評価
- お気に入り
- Patternテンプレート
- Pattern履歴
- AIによるPattern提案
- Plugin Condition
