第1章 システム概要
1.1 目的

本システムは、ゲームとしてファイル整理を行うためのシステムである。

ゲーム中は実際のファイル操作を一切行わず、プレイヤーの操作内容を「変更予定」として記録する。

ゲーム終了後に変更内容をJSONへ出力し、専用の実行プログラムがそのJSONを読み込み、実際のファイル操作を行う。

これにより、

誤操作防止
操作の取り消し
安全性
拡張性

を実現する。

設定内容はJSONとして保存する。

ゲーム起動時に読み込まれる。

実際のファイル変更はこのプロジェクトのみ行う。

1.3 基本方針
実ファイルは変更しない

ゲーム中は実ファイルへ一切アクセスしない。

変更内容はActionとして保持する。

最後に一括実行

ゲーム終了後、

Result JSON

を出力する。

実行プロジェクトがこれを読み込み、一括で処理を行う。

Undo可能

ゲーム中は全て変更予定であるため、

変更
保留
キャンセル

を自由に行える。

AIによる自動整理は行わない

Patternは

提案

のみ行う。

整理を決定するのはプレイヤーである。

1.4 データ管理方針

すべてのデータはJSONで管理する。

データは用途ごとに分割保存する。

設定変更後は次回起動時に反映する。

ゲーム中に設定の動的変更は行わない。

1.5 Result管理

1プレイにつき1つのResultファイルを生成する。

Resultには

作成フォルダ
Action一覧
メタ情報
実行順

を保存する。

1.6 ID管理

システム内で管理する全てのデータは一意のIDを持つ。

対象

File
Folder
Pattern
Skill
Plugin
Theme
Replay

名前ではなくIDで管理する。

1.9 基本設計思想

本システムは

「ゲームとして楽しめるファイル整理」

を目的とする。

ゲーム性は演出・UI・スコアに限定し、

整理内容の判断はプレイヤーが行う。

AIやPatternは補助機能として設計する。
