# 第6章 保存システム（Save System）

## 6.1 概要

保存システムは、本ソフトで使用するすべてのデータの保存・読込を管理するシステムである。

ゲームの進行状況だけではなく、設定・Pattern・Skill・統計・Replayなども本システムで管理する。

保存データは用途ごとに分割し、JSON形式で保存する。

---

# 6.2 基本方針

保存システムは以下の方針で設計する。

- すべてJSON形式
- 用途ごとにファイルを分割
- 人が読める形式
- 編集可能
- バックアップしやすい
- 拡張しやすい
- バージョン管理可能

---

- テーマ

- 保存先

---

# 6.5 オートセーブ

ゲーム中は一定間隔で

Sessionを保存する。

オートセーブ間隔は

設定から変更できる。

オートセーブを無効化できる。

---

# 6.6 バックアップ

保存前に

バックアップを作成できる。

バックアップ数は

設定から変更できる。

古いバックアップは自動削除できる。

---

# 6.7 保存方式

用途ごとにファイルを分割する。

例

Config/

Pattern/

Skill/

Replay/

Statistics/

Result/

Session/

Plugin/

Theme/

Appearance/

---

# 6.8 データ読込

ゲーム起動時

以下の順番で読み込む。

1. Config

2. Theme

3. Appearance

4. Plugin

5. Skill

6. Pattern

7. Statistics

8. Session

Sessionは存在する場合のみ読み込む。

---

# 6.9 Session

Sessionはゲーム途中の状態を保存する。

保存内容

- プレイヤー位置
- 現在フォルダ
- 整理状態
- ActionQueue
- Focus
- Combo
- Score
- Time

Sessionからゲームを再開できる。

---

# 6.10 Result

Resultは

1プレイにつき1ファイル保存する。

Resultには

- メタ情報
- Action
- CreateFolder
- Replay参照
- プレイ結果

を保存する。

実ファイル変更は行わない。

---

# 6.11 Statistics

Statisticsには

累計データを保存する。

例

- 総プレイ時間
- 総整理数
- Pattern利用回数
- Skill利用回数
- ベスト記録

---

# 6.12 Replay

Replayは

ゲーム中の操作履歴を保存する。

保存対象

- プレイヤー移動
- Action
- Skill使用
- Pattern使用
- Focus変化

Replayは閲覧専用とする。

---

# 6.13 バージョン管理

保存データは

Versionを持つ。

ゲーム起動時

Versionを確認し、

必要に応じて変換処理を行う。

---

# 6.14 保存失敗

保存失敗時は

データを上書きしない。

バックアップが存在する場合は

バックアップを保持する。

エラー内容をログへ記録する。

---

# 6.15 読込失敗

JSON読込失敗時

対象データのみ初期化する。

他データは読み込みを継続する。

ゲーム全体を停止しない。

---

# 6.16 初期化

以下を個別に初期化できる。

- UI
- Theme
- Skill
- Pattern
- Statistics
- Replay
- Session

全初期化にも対応する。

---

# 6.17 インポート

以下を外部から読み込める。

- Pattern
- Pattern Pack
- Skill
- Theme
- Plugin
- Appearance

読込時は

Versionを確認する。

---

# 6.18 エクスポート

以下を出力できる。

- Pattern
- Pattern Pack
- Skill
- Theme
- Statistics
- Replay

共有可能な形式で保存する。

---

# 6.19 保存先

保存先フォルダは

設定プロジェクトから変更できる。

ゲーム起動時に設定を読み込む。

変更後は次回起動時に反映する。

---

# 6.20 セキュリティ

保存データは暗号化しない。

プレイヤーが自由に編集できることを優先する。

不正なJSONはエラーとして扱う。

---
